# Content Guidelines & Placeholders

## Overview
This document provides content templates, microcopy guidelines, and placeholder text for the MAANG-ready developer portfolio.

## Content Strategy

### Target Audience
- **Primary**: MAANG recruiters and hiring managers
- **Secondary**: Technical leads and engineering managers
- **Tertiary**: Fellow developers and potential collaborators

### Content Principles
1. **Outcome-focused**: Emphasize measurable impact over technical details
2. **Scannable**: Use bullet points, short paragraphs, and clear hierarchy
3. **Credible**: Include specific metrics and technologies
4. **Professional**: Confident but not boastful tone

## Project Content Templates

### Project Naming Convention
- Keep titles to 3-5 words
- Use descriptive, professional language
- Avoid overly technical jargon

### Project Title Examples
```
✅ Good:
- "Distributed File Pipeline"
- "ML Fraud Detection"
- "Cloud Infrastructure Automation"
- "Real-time Analytics Dashboard"

❌ Avoid:
- "My Awesome ML Project"
- "Super Cool Backend Thing"
- "Project #1"
```

### Tagline Templates (8-12 words)
```
Format: [Technology/Approach] for [Business Value/Use Case]

Examples:
- "Scalable ingestion & processing for telemetry data"
- "Real-time anomaly detection for financial transactions"
- "Infrastructure as Code for multi-region deployment"
- "Live data visualization for business intelligence"

### Impact Bullets (12-18 words each)
```

## Tone & Voice Guidelines

### Personality Traits
- **Confident**: Showcase achievements without being boastful
- **Technical**: Use precise technical language appropriately
- **Results-oriented**: Focus on outcomes and impact
- **Professional**: Maintain formal but approachable tone

### Writing Style
- Use active voice
- Write in first person for personal sections
- Use present tense for current work, past tense for completed projects
- Keep sentences concise and scannable
- Use parallel structure in lists

### What to Avoid
- Overly casual language ("awesome", "cool", "amazing")
- Vague statements without metrics
- Technical jargon without context
- Lengthy paragraphs
- Passive voice constructions
